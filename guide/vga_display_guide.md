# Documentation: `vga_display.v` Implementation Guide

## 1. Overview & Objective

Implement `vga_display.v` — the **VGA display controller** that reads pixel data from BRAM and drives the VGA output pins.

**Dependencies**: `types.vh`, `hsl2rgb.vh`, and one of `vga_gen_640x480.v` / `vga_gen_800x600.v` / `vga_gen_1024x768.v` (selected by macro in `types.vh`)

---

## 2. Port Interface

```verilog
module vga_display (
    input  wire vga_clk_in,           // VGA pixel clock
    input  wire rst_in,
    input  wire [3:0]  read_data_in,  // 4-bit grayscale pixel from BRAM
    input  wire        toggle_hue,    // enable animated hue
    input  wire        toggle_color,  // enable saturation
    output reg  [`ADDR_BITS-1:0] read_addr_out,  // address to read from BRAM
    output reg  [3:0]  vga_r, vga_g, vga_b,      // VGA color outputs (4-bit each)
    output reg         vga_hs, vga_vs             // VGA sync signals
);
```

---

## 3. Architecture: 2-Cycle Pipeline

The BRAM read has a 2-cycle latency. To compensate, the VGA sync signals (`hsync`, `vsync`, `blank`) and the pixel address are pipelined by 2 clock cycles so they arrive at the output at the same time as the data.

```
Cycle 0: hcount/vcount generated by VGA gen
          → request read_addr_out from BRAM using these coordinates
          → pipeline hcount, vcount, hsync, vsync, blank

Cycle 1: (BRAM reading internally)
          → mid-stage pipeline registers

Cycle 2: read_data_in arrives from BRAM
          → delayed sync signals arrive too
          → convert data to color via hsl2rgb
          → drive vga_r/g/b/hs/vs
```

---

## 4. VGA Timing Generator Submodule

Instantiate the VGA timing generator using the macro `\`VGA_GEN_TYPE` (resolves to one of the vga_gen modules):

```verilog
`VGA_GEN_TYPE vga_gen_inst (
    .pixel_clk_in(vga_clk_in),
    .hcount_out(hcount),
    .vcount_out(vcount),
    .vsync_out(vsync),
    .hsync_out(hsync),
    .blank_out(blank)
);
```

---

## 5. Hue Animation Counter

A 28-bit counter increments each VGA clock cycle when `toggle_hue` is high:

```verilog
reg [27:0] hue_counter;
always @(posedge vga_clk_in) begin
    if (rst_in) hue_counter <= 0;
    else hue_counter <= hue_counter + toggle_hue;
end
```

`hue_counter >> 20` gives a slowly changing 8-bit hue value (upper 8 bits of the 28-bit counter).

---

## 6. Color Conversion

Call `hsl2rgb` to convert the pixel's grayscale intensity to a colored output:

```verilog
wire [23:0] hsl;
assign hsl = hsl2rgb(
    hue_counter >> 20,                          // hue: slow animation
    toggle_color ? 8'd165 : 8'd0,               // saturation: 165 (colorful) or 0 (gray)
    read_data_in << 4                            // lightness: 4-bit pixel → 8-bit range
);
// hsl[23:16] = Red, hsl[15:8] = Green, hsl[7:0] = Blue
```

---

## 7. Pipeline Registers

```verilog
// Declare both stages for each signal
reg [`VGA_H_BITS-1:0] hcount, hcount_mid, hcount_out_r;
reg [`VGA_V_BITS-1:0] vcount, vcount_mid, vcount_out_r;
reg vsync, vsync_mid, vsync_out_r;
reg hsync, hsync_mid, hsync_out_r;
reg blank, blank_mid, blank_out_r;

// One always block pipelines all of them
always @(posedge vga_clk_in) begin
    hcount_mid <= hcount;     hcount_out_r <= hcount_mid;
    vcount_mid <= vcount;     vcount_out_r <= vcount_mid;
    vsync_mid  <= vsync;      vsync_out_r  <= vsync_mid;
    hsync_mid  <= hsync;      hsync_out_r  <= hsync_mid;
    blank_mid  <= blank;      blank_out_r  <= blank_mid;
end
```

---

## 8. BRAM Address and Output Logic

```verilog
always @(posedge vga_clk_in) begin
    // Request address (cycle 0): scale VGA pixel coords down to render resolution
    read_addr_out <= ((vcount >> `DISPLAY_SHIFT_SIZE) * `DISPLAY_WIDTH)
                   + (hcount >> `DISPLAY_SHIFT_SIZE);

    // Drive outputs (cycle 2): use delayed sync signals
    vga_r <= blank_out_r ? 4'b0 : hsl[23:20]; // top 4 bits of red
    vga_g <= blank_out_r ? 4'b0 : hsl[15:12]; // top 4 bits of green
    vga_b <= blank_out_r ? 4'b0 : hsl[7:4];   // top 4 bits of blue
    vga_hs <= ~hsync_out_r;  // sync polarity is inverted on this board
    vga_vs <= ~vsync_out_r;
end
```

> **Key note**: `hsl[23:16] >> 4` takes the top 4 bits. You can write this as `hsl[23:20]` directly for clarity.
